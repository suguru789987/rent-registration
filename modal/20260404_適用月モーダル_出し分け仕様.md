# 適用月設定モーダル 出し分け仕様

## 概要

地代家賃の親設定保存時に表示される適用月設定モーダルを、操作と単月設定の状態に応じて3パターンに出し分ける。
予算期間設定機能がリリースされるまでの暫定対応として、予算開始月プルダウンを設置し、過去月への誤適用を防止する。

## 出し分け条件

| # | 操作 | 条件 | 表示UI |
|---|------|------|--------|
| 1 | 親設定を新規作成 | 新しい適用開始日 | プルダウン付き確認ダイアログ |
| 2 | 親設定を編集保存 | 単月設定0件 | プルダウン付き確認ダイアログ（開始月は前回値） |
| 3 | 親設定を編集保存 | 単月設定1件以上 | 単月設定件数警告 + 月別カードモーダル |

### 判定ロジック

```
if (新規作成) {
  → パターン1: 初回登録ダイアログ
} else if (単月設定件数 === 0) {
  → パターン2: 再更新ダイアログ（単月なし）
} else {
  → パターン3: 再更新モーダル（単月あり）
}
```

## パターン1: 初回登録時（ダイアログ）

### 表示内容

- タイトル: 「デフォルト計算額の適用」
- サブタイトル: 「賃料設定を各月に適用します」
- 賃料設定条件ボックス（段階別歩合 / 最大値選択 / 固定額 / 最低保証 / 適用開始日）
- **予算開始月プルダウン**（適用開始月以降のみ選択可）
- メッセージ表示エリア
- ボタン: 「← 費用設定に戻る」「キャンセル」「適用する」

### 動作

- **初期状態**: プルダウン未選択、「予算開始月を選択してください」（赤字）、適用ボタンは活性だがバリデーションあり
- **未選択で適用ボタン押下**: 「⚠ 予算開始月が選択されていません。予算開始月を選択してから適用してください。」（赤字アラート）、プルダウン枠が赤くハイライト
- **選択後**: 「{N}月〜12月にデフォルト計算額を適用します」（緑字）
- **適用ボタン押下**: 選択した開始月〜12月にデフォルト計算額を適用して保存

## パターン2: 再更新時・単月設定なし（ダイアログ）

### 表示内容

- パターン1とほぼ同じ構成
- タイトル: 「デフォルト計算額の更新適用」
- サブタイトル: 「賃料設定を更新し、各月に適用します」
- 新しいデフォルト計算額を表示
- **予算開始月プルダウン**: 前回設定した値がデフォルト選択済み

### 動作

- パターン1と同様（ただしプルダウンは前回値で初期選択済み）

## パターン3: 再更新時・単月設定あり（モーダル）

### 表示内容

- タイトル: 「デフォルト計算額の適用確認」
- サブタイトル: 「各月の設定を活かすか、デフォルト計算額で上書きするか選択してください。」
- **警告バー**: 「⚠ 単月地代家賃の登録が ◯件 あります」（オレンジ背景）
- 新しいデフォルト計算額
- **予算開始月プルダウン**: 前回設定した値がデフォルト選択済み、変更可能
- 月別カードリスト（開始月〜12月）

### 月別カードの状態

| 状態 | 初期チェック | 表示 | クリック動作 |
|------|------------|------|------------|
| 単月設定済み | OFF | オレンジ枠、「単月設定済」タグ、現在の設定額、「→ 単月設定を維持」 | ONにすると「→ デフォルトで上書き」に変化 |
| 単月設定なし | ON | 緑枠、デフォルト計算額、「→ デフォルト適用」 | OFFにすると「→ 変更なし」に変化 |

### 動作

- プルダウン変更 → 表示月が開始月〜12月に更新、単月設定件数も連動
- 個別カードのチェックON/OFF → 各月ごとに適用/維持を選択
- 反映ボタン → チェックONの月にデフォルト計算額を適用、チェックOFFの月は現状維持

## 予算開始月プルダウン共通仕様

- 選択肢: 親設定の適用開始月以降のみ
- 終了月: **12月固定**（予算期間設定機能リリースまでの暫定）
- 選択 → 開始月〜12月が適用対象

## 段階リリース計画

| Phase | 内容 |
|-------|------|
| Phase 1（今回） | 3パターン出し分け + 予算開始月プルダウン（終了月12月固定） |
| Phase 2（次回） | 予算期間設定機能をリリース |
| Phase 3（統合） | プルダウンを削除 or 予算期間未設定時のフォールバック（アラートメッセージのみでも可） |
