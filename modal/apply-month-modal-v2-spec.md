# 適用月設定モーダル v2 仕様（プルダウン削除版）

## 概要

地代家賃の親設定保存時に表示される適用月設定モーダルを、単月設定の有無に応じて2パターンに出し分ける。
予算開始月プルダウンを廃止し、親設定の適用開始日から適用範囲を自動判定する。

## v1からの変更点

- 予算開始月プルダウンを削除（適用開始日から自動判定）
- 初回登録と再更新（単月なし）を同一ダイアログに統合
- 3タブ → 2タブに簡素化

## 出し分け条件

| # | 条件 | 表示UI |
|---|------|--------|
| A | 単月設定なし（初回登録 or 再更新） | 確認ダイアログ |
| B | 単月設定1件以上（再更新） | 月別カードモーダル |

### 判定ロジック

```
if (単月設定件数 === 0) {
  → パターンA: 確認ダイアログ
} else {
  → パターンB: 月別カードモーダル
}
```

## 適用範囲の自動判定

- 適用開始日の月〜12月が適用範囲（終了月は12月固定）
- 例：適用開始日 2026-03-01 → 3月〜12月に適用
- 予算開始月プルダウンは不要（適用開始日 = 予算開始月と見なす）

## パターンA: 確認ダイアログ（単月設定なし）

### 表示内容

- タイトル: 「デフォルト計算額の適用」
- サブタイトル: 「賃料設定を各月に適用します」
- 賃料設定条件ボックス（段階別歩合 / 最大値選択 / 固定額 / 最低保証 / 適用開始日）
- メッセージ: 「{N}月〜12月にデフォルト計算額を適用します」（緑字）
- ボタン: 「← 地代家賃設定に戻る」「キャンセル」「適用する」

### 動作

- 適用ボタン押下 → 適用開始月〜12月にデフォルト計算額を適用して保存

## パターンB: 月別カードモーダル（単月設定あり）

### 表示内容

- タイトル: 「デフォルト計算額の適用確認」
- サブタイトル: 「各月の設定を活かすか、デフォルト計算額で上書きするか選択してください。」
- 警告バー: 「⚠ 単月地代家賃の登録が ◯件 あります」（オレンジ背景）
- 新しいデフォルト計算額 + 適用範囲表示
- 月別カードリスト（適用開始月〜12月）
- ボタン: 「← 地代家賃設定に戻る」「キャンセル」「反映する」

### 月別カードの状態

| 状態 | 初期チェック | 表示 | クリック動作 |
|------|------------|------|------------|
| 単月設定済み | OFF | オレンジ枠 /「単月設定済」タグ /「→ 単月設定を維持」 | ONにすると「→ デフォルトで上書き」に変化 |
| 単月設定なし | ON | 緑枠 /「→ デフォルト適用」 | OFFにすると「→ 変更なし」に変化 |

### 動作

- 個別カードのチェックON/OFF → 各月ごとに適用/維持を選択
- 反映ボタン → チェックONの月にデフォルト計算額を適用、チェックOFFの月は現状維持

## 段階リリース計画

- Phase 1（今回）：2パターン出し分け。単月設定なし→確認ダイアログ / 単月設定あり→月別カードモーダル。適用範囲は適用開始月〜12月固定（予算開始月プルダウンなし、適用開始日から自動判定）
- Phase 2（次回）：予算期間設定機能をリリース。適用範囲の終了月を予算期間終了月に変更
- Phase 3（統合）：確認ダイアログを廃止（予算期間設定により適用範囲が自動確定するため）。単月設定がある場合の月別カードモーダルのみ残す
