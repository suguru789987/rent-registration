# 適用月設定モーダル：予算開始月プルダウン暫定仕様

## 背景・目的

地代家賃の親設定保存時に表示される「適用月設定モーダル」において、現状1月〜12月の全月がチェックボックスで表示され、すべてチェックON状態になっている。
これにより、予算が存在しない過去月にも地代家賃が適用され、意図しない上書きが発生する問題がある。

本仕様は、**予算期間設定機能がリリースされるまでの暫定対応**として、適用月モーダルに予算開始月プルダウンを追加し、過去月への誤適用を防止する。

## 現状の問題

- 親設定保存時のモーダルで1月〜12月すべてにチェックが入っている
- 予算が未設定の月にもデフォルト計算額が適用されてしまう
- 過去月の地代家賃や予算が上書きされるリスクがある

## 暫定仕様

### 1. 「すべて選択」を予算開始月プルダウンに変更

- 現在の「すべて選択」チェックボックスの位置に、予算開始月プルダウンを配置する
- 予算開始月を選択すると、**開始月〜12月がすべてチェックON状態**で表示される
- 不要な月はチェックを外して個別に除外可能
- 開始月より前の月は非表示 or 非活性
- 終了月は**12月固定**（予算期間設定機能がないため）

### 2. 赤メッセージ表示

- **初期状態**：「予算開始月を選択してください」（赤字、反映ボタンは非活性）
- **選択後**：「{N}月〜12月にデフォルト計算額を適用します」（緑字に変化）
- 反映前に何が起きるか明確にする

### 3. 適用開始日との関係

- プルダウンの選択肢は、親設定の**適用開始月以降のみ**に制限する
- 例：適用開始日が2026-03-01 → プルダウンに1月・2月は表示しない

## 段階リリース計画

| Phase | 内容 |
|-------|------|
| Phase 1（今回） | 適用月モーダルの「すべて選択」を予算開始月プルダウンに変更 + 赤メッセージ（終了月は12月固定） |
| Phase 2（次回） | 予算期間設定機能をリリース |
| Phase 3（統合） | プルダウンを削除 or 予算期間未設定時のフォールバック（アラートメッセージのみでも可） |

## 注意事項

- 本仕様はあくまで**過去月の誤上書き防止**が目的の暫定機能である
- 終了月の制御は予算期間設定機能で対応する
- 予算期間設定機能リリース後、本プルダウンは削除 or フォールバックとして残す
